# 人物与番剧简称（LLM）

本目录用 LLM 为「角色名 + cosplay + 番名」类搜索生成**常用简称**，便于图片搜索命中。

## 背景

实际检索时常用简称，例如：

- `"攻殻機動隊 S.A.C. 2nd GIG "` → `"攻壳机动队"`
- `"惣流·明日香·兰格雷"` → `"明日香"`
- `"阿尔托莉雅·潘德拉贡"` → `"阿尔托莉雅"`
- `"艾伦·耶格尔"` → `"艾伦"`

角色名与番剧名都希望得到合适简写。

## Prompt 模板

```
请根据给定的角色和番剧信息，以json形式向我返回大家最能接受的角色简称和番剧简称（如有）

简称映射的例子:
"攻殻機動隊 S.A.C. 2nd GIG " --> "攻壳机动队"
"惣流·明日香·兰格雷" --> "明日香"
"新世纪福音战士" --> "EVA"
"阿尔托莉雅·潘德拉贡" --> "阿尔托莉雅"
"艾伦·耶格尔" --> "艾伦"
"初音未来" --> "初音未来"
（如果有中文字使用简体中文）

需要获取简称和番剧简称的角色为
{这里根据character_ranked中的信息，尤其要给出角色的name name_cn 以及出场的所有番剧}

以JSON形式返回你的分析和结果，并包含以下所有字段
- decide_if_brief 简单分析 是否要对角色和番剧进行简称，如果原来的番剧名或者角色名已经足够简洁，可以进行直接沿用
- analysis 大家最常用的简称是哪些，大家一般怎么称呼这个番剧和角色，使用简称的角色*番剧是不是能够基本定位到这个角色
- brief_name 角色的简称
- brief_bangumi 番剧的简称
```

## 环境与实验

- 在 `.env` 中配置 OpenAI 兼容接口的 **API Key** 与 **Base URL**（变量名与 `llm_client.py` 一致）
- 选用**相对经济**的模型
- **不要**在代码中硬编码 base URL 与 API key
- 结果缓存到 `local_data/` 的约定文件中
- 在 **top 1000** 角色中**随机 5 个**做实验
- 实验结果以 **Markdown** 输出到 `information/` 目录
